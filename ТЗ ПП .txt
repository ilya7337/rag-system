# Техническое задание


## 1. Общие положения


### 1.1. Назначение системы
Система представляет собой веб-приложение, предназначенное для накопления, структурирования и поиска знаний в области проектной деятельности, методологий разработки, технологий и развертывания программного обеспечения. Основные пользователи — студенты различных курсов и направлений. Система обеспечивает быстрый доступ к информации через чат-бот  на основе векторного поиска по загруженным документам.


### 1.2. Ролевая модель
В системе предусмотрены две роли:
- **Администратор** — управление контентом через административную панель.
- **Обычный пользователь** (авторизованный и неавторизованный) — взаимодействие с чат-ботом и просмотр материалов.



## 2. Требования к логике работы системы


### 2.1. Управление базой знаний
- Администратор через панель управления загружает документы, относящиеся к определённым темам (карточкам знаний).
- Каждый загруженный документ подлежит векторизации и помещению в векторную базу данных для последующего семантического поиска.


### 2.2. Обработка пользовательских запросов (чат-бот)
- Пользователь вводит текстовый запрос.
- Система векторизует запрос и выполняет поиск наиболее релевантного фрагмента (или нескольких фрагментов) в векторной базе документов.
- Найденные фрагменты передаются в легковесную языковую модель (LLM), которая формирует естественно-языковой ответ. Ответ не должен быть дословным извлечением из документа, а представлять собой адаптированное и связное сообщение.
- Поиск ограничен исключительно документами, загруженными в систему; внешние источники не используются.


### 2.3. Оценка ответов (только веб-интерфейс)
- Пользователь может оценить каждый ответ чат-бота с помощью кнопок «Лайк» и «Дизлайк».
- Доступна функция копирования текста ответа в буфер обмена.


### 2.4. История диалогов
- Все запросы и ответы сохраняются.
- История группируется по дате: запросы, сделанные в разные календарные дни, не могут находиться в одной карточке истории.
- Предусмотрена кнопка «Новый чат»: создание нового чата приводит к формированию отдельной карточки в истории.
- В интерфейсе истории каждый диалог отображается с меткой (например, первый запрос или краткое содержание). При наведении на метку отображается начало первого запроса диалога. По клику на метку осуществляется переход к соответствующему диалогу.


### 2.5. Обработка неавторизованных пользователей
- Неавторизованный пользователь имеет доступ к чат-боту на сайте, но его история запросов не сохраняется в интерфейсе.
- При этом все его запросы и ответы фиксируются в системе (для административной панели) обезличенно.


## 3. Требования к интерфейсу сайта


### 3.1. Общая структура
Сайт является многостраничным. Страницы генерируются на основе карточек тем (разделов знаний), созданных администратором. Каждая карточка темы (например, «Agile») при клике открывает отдельную страницу, содержащую все документы и данные, относящиеся к этой теме.


### 3.2. Главная страница
- Приветственный текст.
- Сетка (грид) из всех карточек тем, созданных администратором.
- В правом верхнем углу для неавторизованного пользователя отображаются кнопки **«Регистрация»** и **«Вход»**.


### 3.3. Боковая панель чат-бота
- В правом верхнем углу (или фиксированный элемент) размещается иконка чат-бота.
- При клике на иконку справа открывается боковая панель шириной в половину страницы (или фиксированной ширины) с интерфейсом чат-бота.
- При открытой панели пользователь может продолжать взаимодействие с основным содержимым страницы (прокрутка, переход по карточкам и т.д.).
- В боковой панели по умолчанию отображается поле ввода запроса и история диалогов (занимает треть ширины панели). История может быть скрыта пользователем (кнопка «Закрыть/крестик») и повторно открыта через соответствующую кнопку.
- Кнопка «Новый чат» очищает текущий диалог в панели и создаёт новую карточку в истории.


### 3.4. Форма обратной связи
- Кнопка вызова формы расположена в правом нижнем углу экрана.
- Пользователь оставляет только текстовый комментарий.
- Комментарий сохраняется в системе обезличенно.


## 4. Требования к авторизации и регистрации


### 4.1. Регистрация
- Поля: **Логин** (уникальное значение), **Пароль**.
- Требования к паролю: минимальная длина 8 символов, обязательное наличие цифр.


### 4.2. Вход
- Выполняется по логину и паролю.


## 5. Требования к административной панели


Административная панель состоит из трёх разделов (страниц).


### 5.1. Раздел «База знаний»
- **Управление карточками тем:**
    - Создание темы с полями: *Заголовок* (обязательное), *Описание* (обязательное).
    - Редактирование существующих тем.
- **Управление документами внутри темы:**
    - При переходе в созданную карточку темы администратор может нажать кнопку «Загрузить документ».
    - Открывается модальное окно с полями:
        - *Название* (обязательное)
        - *Описание* (опциональное)
        - *Файл документа* (загрузка из локального хранилища)
    - После загрузки документ автоматически векторизуется и индексируется в векторной базе данных.


### 5.2. Раздел «Запросы»
- Отображается полный журнал всех запросов и ответов системы (обезличенно).
- Каждая запись (сессия/диалог) представлена в виде карточки, аналогичной FAQ-блоку.
- Карточка в свёрнутом виде показывает первый запрос пользователя.
- По нажатию на кнопку «Подробнее» карточка раскрывается и отображает полную историю диалога: все последующие вопросы пользователя и ответы бота.
