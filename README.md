# База знаний с AI-чатом (FastAPI MVP)

Веб-приложение на Django, которое хранит решения ФАС, строит эмбеддинги текста, выполняет семантический поиск по косинусной близости в PostgreSQL с pgvector и отображает результаты в виде чат-интерфейса. Пользователь вводит запрос, сервис получает эмбеддинг запроса, ищет ближайшие решения по векторной метрике и возвращает подборку наиболее релевантных документов.

Приложение принимает решения через API, сохраняет метаданные и текст PDF, строит эмбеддинги для поиска и выдаёт результаты по смысловой близости. Поиск реализован через pgvector и CosineDistance, а для ускорения используется HNSW-индекс. В выдаче дополнительно может формироваться краткое содержание решения на основе полного текста с помощью LLM.

---


## Структура репозитория

В проекте используется следующая организация каталогов.

`src/api/internal/decisions/` содержит модуль решений: модели, DTO, сервис, обработчики и транспорт.  
`src/frontend/` содержит статический интерфейс: `index.html`, `style.css`, `app.js`.  
`docker/` содержит Dockerfile и конфигурацию для app, nginx и postgres.  
`docker-compose.yml` поднимает весь стек.

## Конфигурация

Для работы необходимы параметры доступа к LLM.

`OPENAI_API_KEY` используется OpenAI-клиентом.  
`OPENAI_CLIENT_BASE_URL` задаёт базовый URL OpenAI-клиента, если используется не стандартный endpoint.

## Запуск

1. Создать в корневой директории файл `.env` и скопировать в него содержимое `.env.example`
2. Заполнить переменные окружения
3. `make all` - запустить сборку
4. `make createsuperuser` - создать суперпользователя для использования административной панели
5. `make scrape start_page= end_page=` - выгрузить на сервер решения с сайта ФАС, начиная со страницы `start_page`, заканчивая страницей `end_page`
6. После успешного запуска перейти на http://localhost:1345/. 
    Административная панель: http://localhost:1345/admin/. 
    Сваггер: http://localhost:1345/api/docs.

---

## API

Сервис предоставляет HTTP API для работы с решениями.

`POST`-метод добавляет решение и сохраняет эмбеддинг текста.  
`GET`-метод списка принимает строку поиска и возвращает подборку решений по смысловой близости.  

Клиентская часть использует `GET` для поиска, передавая `search_string`

---

## Frontend

Интерфейс реализован как чат-окно. Пользователь вводит запрос в поле ввода и отправляет его, после чего в чат добавляется сообщение пользователя и сообщение бота с результатами поиска.

Файлы фронтенда расположены в `src/frontend/`:
- `index.html` подключает `/static/style.css` и `/static/app.js`
- `style.css` описывает внешний вид чата и карточек решений
- `app.js` выполняет запросы к API и рендерит результаты

Быстрые кнопки подставляют текст в поле ввода без автоматической отправки, что работает как автоподсказка для типовых запросов.
