# База знаний с AI-чатом (FastAPI)

Веб-приложение на FastAPI, предназначенное для накопления, структурирования и поиска знаний в области проектной деятельности, методологий разработки, технологий и развертывания программного обеспечения, строит эмбеддинги текста, выполняет семантический поиск по косинусной близости в PostgreSQL с pgvector и отображает результаты в виде чат-интерфейса. Пользователь вводит запрос, сервис получает эмбеддинг запроса, ищет ближайшие решения по векторной метрике и возвращает подборку наиболее релевантных документов.

Приложение принимает решения через API, сохраняет метаданные и текст PDF, строит эмбеддинги для поиска и выдаёт результаты по смысловой близости. Поиск реализован через pgvector и CosineDistance, а для ускорения используется HNSW-индекс. В выдаче дополнительно может формироваться краткое содержание решения на основе полного текста с помощью LLM.

## Структура репозитория

В проекте используется следующая организация каталогов.

- `src/` — основной код приложения
- `src/routers/` — API роутеры (auth, chat, documents, feedback, topics, admin)
- `src/services/` — бизнес-логика (LLM, PDF)
- `src/static/` — статические файлы фронтенда
- `docker/` — Dockerfile и конфигурация для app, nginx и postgres
- `docker-compose.yml` — поднимает весь стек

## Конфигурация

Для работы необходимы параметры доступа к LLM и базы данных.

### Переменные окружения

Создайте файл `.env` в корневой директории со следующими переменными:

```bash
SECRET_KEY=your-secret-key-here
DATABASE_URL=postgresql+asyncpg://user:password@localhost/db
POSTGRES_USER=knowledge_user
POSTGRES_PASSWORD=knowledge_pass
POSTGRES_DB=knowledge_db
OPENAI_API_KEY=your-openai-api-key
OPENAI_BASE_URL=https://routerai.ru/api/v1
EMBEDDING_MODEL=baai/bge-m3
LLM_MODEL=openai/gpt-oss-120b
DEBUG=true
```

### Параметры

- `SECRET_KEY` — секретный ключ для подписи токенов
- `DATABASE_URL` — URL для подключения к PostgreSQL с pgvector
- `OPENAI_API_KEY` — API ключ OpenAI
- `OPENAI_BASE_URL` — базовый URL API (по умолчанию: https://routerai.ru/api/v1)
- `EMBEDDING_MODEL` — модель для создания эмбеддингов (по умолчанию: baai/bge-m3)
- `LLM_MODEL` — LLM модель для генерации (по умолчанию: openai/gpt-oss-120b)
- `DEBUG` — включить режим отладки

## Запуск

1. Создать в корневой директории файл `.env` и заполнить переменные окружения
2. `make build` — собрать образы
3. `make up` — запустить контейнеры
4. `make create-admin login=admin password=your-password` — создать администратора
5. После успешного запуска перейти на http://localhost:8000/. 
   Административная панель: http://localhost:8000/admin/. 
   Сваггер: http://localhost:8000/docs.

## API

Сервис предоставляет HTTP API для работы с решениями.

### Основные эндпоинты

- `POST /api/documents` — добавить решение и сохранить эмбеддинг текста
- `GET /api/documents` — получить список решений с поиском по смысловой близости
- `GET /api/chat` — поиск с возвратом результатов в формате чата
- `POST /api/auth/login` — аутентификация пользователя
- `POST /api/auth/register` — регистрация нового пользователя

Клиентская часть использует `GET /api/chat` для поиска, передавая параметр `query`

## Frontend

Интерфейс реализован как чат-окно. Пользователь вводит запрос в поле ввода и отправляет его, после чего в чат добавляется сообщение пользователя и сообщение бота с результатами поиска.

Файлы фронтенда расположены в `src/static/`:
- `index.html` — главная страница, подключает стили и скрипты
- `style.css` — описывает внешний вид чата и карточек решений
- `app.js` — выполняет запросы к API и рендерит результаты

Быстрые кнопки подставляют текст в поле ввода без автоматической отправки, что работает как автоподсказка для типовых запросов.
